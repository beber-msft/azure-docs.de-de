# <a name="azure-stack-edge-2110-release-notes"></a>Versionshinweise zu Azure Stack Edge 2110

[!INCLUDE [applies-to-GPU-and-pro-r-and-mini-r-skus](../../includes/azure-stack-edge-applies-to-gpu-pro-r-mini-r-sku.md)]

In den folgenden Versionshinweisen werden schwerwiegende ungelöste und behobene Probleme für das Release 2110 für Azure Stack Edge-Geräte beschrieben. Diese Versionshinweise gelten nur für Geräte vom Typ „Azure Stack Edge Pro – GPU“, „Azure Stack Edge Pro R“ und „Azure Stack Edge Mini R“. Funktionen und Probleme, die nur für ein bestimmtes Modell gelten, werden jeweils entsprechend gekennzeichnet.

Die Versionshinweise werden fortlaufend aktualisiert, und wenn schwerwiegende Probleme festgestellt werden, die eine Problemumgehung erfordern, werden sie hinzugefügt. Lesen Sie vor der Bereitstellung Ihres Geräts die Informationen in den Versionshinweisen sorgfältig durch.

Dieser Artikel bezieht sich auf das **Release 2110 von Azure Stack Edge**, das der Softwareversionsnummer **2.2.1758.4034** zugeordnet ist. Diese Software kann auf Ihr Gerät angewendet werden, wenn Sie mindestens Azure Stack Edge 2106-Software (2.2.1636.3457) betreiben.

## <a name="whats-new"></a>Neues

Im Release 2110 von Azure Stack Edge sind die folgenden neuen Funktionen verfügbar. 

- **Windows Updates und Sicherheitsupdates:** Das [neueste kumulative Update (LCU) für Windows- und September-Sicherheitskorrekturen](https://support.microsoft.com/topic/september-14-2021-kb5005623-monthly-rollup-bcdb6598-517e-4d53-aa7c-dd7fcfdca204) wurde im Updatepaket für Azure Stack Edge eingeführt.
- **Fernbetreuung** – In dieser Version können Sie die Remoteunterstützung auf Ihrem Azure Stack Edge Gerät aktivieren, damit Microsoft-Support Probleme diagnostizieren und beheben können, indem Sie remote auf Ihr Gerät zugreifen. Wenn Sie diese Funktion aktivieren, geben Sie Ihr Einverständnis für die Intensität und die Dauer des Zugriffs. Weitere Informationen finden Sie unter [Aktivieren der Fernbetreuung und -diagnose für Azure Stack Edge](azure-stack-edge-gpu-remote-support-diagnostics-repair.md).
- Virtuelle Computer mit **hohem Leistungsnetzwerk** – Ab diesem Release können virtuelle Netzwerkcomputer mit hoher Leistung auf Ihrem Azure Stack Edge Gerät bereitgestellt werden. Weitere Informationen finden Sie unter [Bereitstellen von virtuellen Netzwerkcomputern](azure-stack-edge-gpu-deploy-virtual-machine-high-performance-network.md) mit hoher Leistung auf Azure Stack Edge.
- **Zertifikate für die Edge-Containerregistrierung und das Kubernetes-Dashboard** – Zertifikate für die Edge-Containerregistrierung und das Kubernetes-Dashboard werden jetzt unterstützt. Sie können Zertifikate über die lokale Benutzeroberfläche erstellen und hochladen. Weitere Informationen finden Sie unter [Kubernetes-Zertifikate](azure-stack-edge-gpu-certificates-overview.md#kubernetes-certificates) und [Kubernetes-Zertifikate Hochladen](azure-stack-edge-gpu-manage-certificates.md#upload-kubernetes-certificates).
- **Metallb im BGP-Modus** – Ab diesem Release können Sie den Lastenausgleich auf Ihrem Azure Stack Edge Gerät mithilfe von MetalLB über Border Gateway Protocol (BGP) konfigurieren. Sie können MetalLB im BGP-Modus konfigurieren, indem Sie eine Verbindung mit der PowerShell-Schnittstelle des Geräts herstellen und dann bestimmte Cmdlets ausführen. Weitere Informationen finden Sie unter [Konfigurieren des Lastenausgleichs mit MetalLB auf Ihrem Azure Stack Edge Gerät](azure-stack-edge-gpu-configure-metallb-bgp-mode.md).



## <a name="issues-fixed-in-2110-release"></a>Im Release 2110 behobene Probleme

In der folgenden Tabelle sind die Probleme aufgelistet, die in vorherigen Releases festgestellt und im aktuellen Release behoben wurden.

| Nein. | Funktion | Problem | 
| --- | --- | --- |
|**1.**|Kubernetes mit Azure Arc-Aktivierung |Im allgemein verfügbaren Release wird Azure Arc-fähiges Kubernetes von Version 0.1.18 auf 0.2.9. Da das Update für Azure Arc-fähiges Kubernetes nicht auf Azure Stack Edge-Geräten unterstützt wird, müssen Sie Azure Arc-fähiges Kubernetes erneut bereitstellen.|
|**2.**|Kubernetes mit Azure Arc-Aktivierung|Azure Arc-Bereitstellungen werden nicht unterstützt, wenn auf dem Azure Stack Edge Pro-Gerät ein Webproxy konfiguriert ist.|
|**3.**|IoT Edge |Über IoT Edge bereitgestellte Module können das Hostnetzwerk nicht verwenden. | 
|**4.**|Kubernetes und Updates |Bei früheren Softwareversionen, z. B. Release 2008, treten bei einem Update Probleme durch eine Racebedingung auf, die beim Update zum Fehler ClusterConnectionException führen. |
|**5.**|Kubernetes-Dashboard | Der *HTTPS*-Endpunkt für das Kubernetes-Dashboard mit SSL-Zertifikat wird nicht unterstützt. | 
|**6.**|VMs |Eine Überprüfung auf Duplizierung der statischen IP-Adresse wurde für die VM-Verwaltungs-NIC während der VNF-Bereitstellung hinzugefügt. Es wird eine explizite Fehlermeldung zurückgegeben. | 
|**7.**|VMs |Die Überprüfung der IP-Reservierung wurde für die ersten vier IP-Adressen im Adressraum entfernt.  | 
|**8.**|Multi-Access Edge Compute |Problem durch Ablauf des lokalen Azure Resource Manager-Tokens während der VNF-Bereitstellung wurde behoben. Wenn in früheren Versionen der VHD-Download lange dauerte, trat bei der VNF-Bereitstellung ein Fehler auf, da das Azure Resource Manager-Token ablief.  | 
|**9.**|Multi-Access Edge Compute |Ein Timeout für Azure Resource Manager-Aufrufe während der VNF-Bereitstellung wurde hinzugefügt. In früheren Versionen dauerte die VNF-Bereitstellung sehr lange, wenn Azure Resource Manager-Aufrufe nicht erfolgreich waren.   | 
|**10.**|Multi-Access Edge Compute |Multi-Access Edge Compute bereinigt Bereitstellungen von Azure Resource Manager-Vorlagen nach Abschluss des VHD-Downloads. In früheren Versionen erhielt der Benutzer nach vielen VNF-Bereitstellungen eine Fehlermeldung, dass das Bereitstellungskontingent überschritten wurde. Das Standardkontingent lag bei 800 Bereitstellungen pro Ressourcengruppe.  |
